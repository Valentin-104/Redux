객체 지향 프로그래밍(Object-Oriented Programming, OOP)은 컴퓨터 프로그래밍의 패러다임 중 하나로, 프로그램을 객체들의 집합으로 파악하고, 프로그래밍을 객체들의 상호작용으로 표현하는 방법이다. 이 접근 방식은 프로그램의 구조와 설계를 더 이해하기 쉽게 만들고, 코드의 재사용성, 확장성, 관리 용이성을 높이는 데 도움이 된다.

### 객체 지향 프로그래밍의 핵심 개념

1. **클래스(Class)**: 객체를 만들기 위한 템플릿 또는 블루프린트다. 클래스는 객체의 구조와 행동을 정의한다.
2. **객체(Object)**: 클래스에 정의된 속성(데이터 필드)과 행동(메소드)을 실제로 가지는 인스턴스다.
3. **상속(Inheritance)**: 한 클래스가 다른 클래스의 속성과 메소드를 상속받을 수 있는 기능이다. 코드 재사용을 용이하게 하고, 계층 구조를 통한 분류가 가능하다.
4. **캡슐화(Encapsulation)**: 객체의 세부 구현 내용을 숨기고, 사용자에게는 필요한 부분만을 인터페이스로 제공하여 객체의 독립성을 보장한다. 이를 통해 객체 내부의 변경이 다른 객체에 영향을 미치지 않도록 한다.
5. **다형성(Polymorphism)**: 같은 이름의 메소드가 다른 클래스에서 다른 작업을 수행할 수 있음을 의미한다. 컴파일 타임(오버로딩)과 런타임(오버라이딩) 다형성이 있다.
6. **추상화(Abstraction)**: 복잡한 실제 상황을 단순화하여 모델링하는 과정이다. 필요한 정보와 기능만을 간추려서 프로그램에 구현하는 것을 의미한다.

객체 지향 프로그래밍은 이러한 개념들을 활용하여 소프트웨어 개발 과정에서 코드의 재사용성을 높이고, 대규모 소프트웨어의 개발 및 유지보수를 용이하게 한다. 대표적인 객체 지향 프로그래밍 언어로는 Java, C++, Python, Ruby 등이 있다.
