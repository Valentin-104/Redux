실행 컨텍스트(Execution Context)는 JavaScript에서 코드가 평가되고 실행되는 환경의 추상적인 개념이다. 이는 현재 실행되고 있는 코드에 대한 정보를 담고 있으며, 변수, 함수 선언, this 키워드 값 등의 정보를 포함한다. 코드를 실행할 때 필요한 모든 정보를 포함하고 있어, JavaScript 엔진이 코드를 어떻게 실행해야 할지 결정하는 데 도움을 준다.

JavaScript 엔진은 실행 컨텍스트를 관리하기 위해 실행 컨텍스트 스택(Execution Context Stack), 종종 호출 스택(Call Stack)이라고도 불리는, 스택 구조를 사용한다. 이 스택의 최상위에 있는 실행 컨텍스트가 현재 실행 중인 컨텍스트다.

### 실행 컨텍스트의 종류

JavaScript에는 주로 세 가지 타입의 실행 컨텍스트가 있다:

1. **전역 실행 컨텍스트(Global Execution Context):** JavaScript 코드가 실행되기 시작할 때 가장 먼저 생성되는 실행 컨텍스트입니다. 전역 변수와 전역 함수를 관리하며, 전역 범위를 대표한다.

2. **함수 실행 컨텍스트(Function Execution Context):** 함수가 호출될 때마다 해당 함수에 대한 새로운 실행 컨텍스트가 생성된다. 이는 함수의 매개변수, 지역 변수를 포함한다.

3. **Eval 실행 컨텍스트(Eval Execution Context):** `eval` 함수가 호출될 때 생성되며, `eval` 함수 내에서 실행되는 코드를 위한 환경을 제공한다.
