Hoisting(호이스팅)은 JavaScript에서 변수 및 함수 선언이 스코프 내의 최상단으로 끌어올려지는 동작을 말한다. 이 동작은 코드 실행 전에 자바스크립트 엔진에 의해 처리되며, 변수와 함수가 선언된 위치와 상관없이 해당 스코프의 최상단으로 끌어올려진다는 특징이 있다.

### 변수 호이스팅:

```javascript
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

위의 코드에서 `console.log(x)`를 호출하기 전에 `var x = 5;`를 선언했지만, 변수 `x`는 선언 위치와 상관없이 스코프의 최상단으로 끌어올려져 `undefined`로 초기화된다. 이후에 변수에 값이 할당되면 해당 값이 나타난다.

### 함수 호이스팅:

```javascript
hello(); // "Hello, World!"

function hello() {
  console.log("Hello, World!");
}
```

위의 코드에서도 함수 `hello`는 호출보다 아래에 정의되었지만, 함수 선언은 스코프의 최상단으로 끌어올려져 호출이 가능하다.

### 주의사항:

1. **선언만 호이스팅, 할당은 제외:**
   
   변수와 함수의 선언만이 호이스팅되며, 할당(초기화) 부분은 호이스팅되지 않는다. 변수의 경우 선언 시 `undefined`로 초기화되고, 함수는 전체가 끌어올려지므로 함수 내용이 모두 호이스팅된다.

2. **let과 const의 블록 스코프:**
   
   `let`과 `const`는 블록 스코프를 가지므로 호이스팅이 발생하지만, 초기화 이전에 접근하면 `ReferenceError`가 발생한다. 이는 변수가 선언되기 전까지 "일시적인 사각지대(Temporal Dead Zone)"에 있다는 것을 의미한다.

    ```javascript
    console.log(y); // ReferenceError: Cannot access 'y' before initialization
    let y = 10;
    ```

호이스팅은 코드의 가독성과 이해를 어렵게 할 수 있으므로, 변수 및 함수는 사용되기 전에 선언하는 것이 권장된다. ES6에서 도입된 `let`과 `const`는 블록 스코프를 가지므로 이러한 문제를 해결하고자 하였다.
