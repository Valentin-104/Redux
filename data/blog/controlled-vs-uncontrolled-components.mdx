React에서 컨트롤된 컴포넌트(Controlled Components)와 비컨트롤된 컴포넌트(Uncontrolled Components)는 폼(form) 엘리먼트를 다루는 두 가지 접근 방식을 나타낸다.

1. **컨트롤된 컴포넌트(Controlled Components):** 이 방식에서는 React 컴포넌트가 폼 데이터의 상태를 관리한다. 입력 폼 엘리먼트(예: `<input>`, `<textarea>`, `<select>`)의 값(value)을 React의 state에 할당하고, 각 폼 엘리먼트의 변경(change) 이벤트를 처리하여, 사용자의 입력을 state에 반영한다. 이렇게 하면 폼의 현재 값이 React의 state에 의해 완전히 제어되므로 "컨트롤된 컴포넌트"라고 한다. 컨트롤된 컴포넌트를 사용하면 폼의 입력 값에 대한 소스가 React 컴포넌트의 state가 되어 React의 데이터 흐름 내에서 일관성을 유지할 수 있다.

2. **비컨트롤된 컴포넌트(Uncontrolled Components):** 이 방식에서는 폼 데이터를 DOM 자체가 관리한다. React에서는 폼 엘리먼트의 초기 값만 설정해주고, 사용자의 입력을 직접 관리하지 않는다. 대신, 필요할 때 (예를 들어, 폼 제출 이벤트가 발생했을 때) DOM에서 폼의 값을 직접 읽어온다. 이 경우 `ref`를 사용하여 폼 엘리먼트에 직접 접근할 수 있다. 비컨트롤된 컴포넌트는 React 외부에서 폼의 상태를 관리하기 때문에, React의 데이터 흐름에 따르지 않는다.

요약하자면, 컨트롤된 컴포넌트는 React state를 통해 폼 입력을 관리하는 방식이고, 비컨트롤된 컴포넌트는 DOM 자체에 폼 데이터를 저장하고 직접 관리하는 방식이다. 컨트롤된 컴포넌트를 사용하면 React의 선언적인 특성과 데이터 흐름을 활용할 수 있지만, 때로는 비컨트롤된 컴포넌트를 사용하여 보다 단순하게 폼 데이터를 처리할 수도 있다.
