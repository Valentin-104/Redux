HTTP 상태 코드는 서버가 클라이언트에게 반환하는 세 자리 숫자로 이루어진 코드이다. 이 코드는 요청이 성공했는지, 실패했는지, 그 이유를 알려주는 역할을 한다. 상태 코드는 다음과 같이 그룹으로 나눌 수 있다:

1. **1xx (Informational - 정보)**
   - 100 Continue: 서버는 요청을 받았고 계속 처리할 준비가 되었다는 의미.
   - 101 Switching Protocols: 서버가 업그레이드 프로토콜을 요청하고 클라이언트가 수락했음을 나타냄.

2. **2xx (Successful - 성공)**
   - 200 OK: 요청이 성공적으로 처리되었음을 나타냄.
   - 201 Created: 요청이 성공적으로 처리되어 새로운 리소스가 생성되었음을 나타냄.
   - 204 No Content: 요청은 성공적으로 처리되었지만 응답 본문에 내용이 없음을 나타냄.

3. **3xx (Redirection - 리디렉션)**
   - 301 Moved Permanently: 리소스가 새로운 위치로 영구적으로 이동되었음을 나타냄.
   - 302 Found: 리소스가 일시적으로 새로운 위치로 이동되었음을 나타냄.
   - 304 Not Modified: 클라이언트의 캐시가 유효하며 리소스가 변경되지 않았음을 나타냄.

4. **4xx (Client Error - 클라이언트 오류)**
   - 400 Bad Request: 요청이 잘못되었음을 나타냄.
   - 401 Unauthorized: 인증이 필요한 리소스에 접근하려고 할 때 사용됨.
   - 403 Forbidden: 요청이 서버에 의해 거부되었음을 나타냄.
   - 404 Not Found: 요청한 리소스가 서버에 없음을 나타냄.

5. **5xx (Server Error - 서버 오류)**
   - 500 Internal Server Error: 서버가 요청을 처리하는 동안 오류가 발생했음을 나타냄.
   - 502 Bad Gateway: 서버가 게이트웨이나 프록시 역할을 하고 있는데, 유효하지 않은 응답을 받았음을 나타냄.
   - 503 Service Unavailable: 서버가 현재 요청을 처리할 수 없음을 나타냄.

상태 코드는 클라이언트와 서버 간의 통신에서 중요한 역할을 하며, 문제를 식별하고 디버깅하는 데 도움이 된다.
