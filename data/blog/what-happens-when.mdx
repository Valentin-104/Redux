브라우저 주소창에 google.com을 입력하고 엔터를 누르면 일어나는 일련의 과정은 다음과 같다. 이 과정은 인터넷에서 정보를 찾고, 화면에 웹 페이지를 표시하는 복잡한 단계들로 구성된다.

### 1. URL 해석

- 브라우저는 입력된 URL이 유효한 웹 주소인지 확인한다. 이 경우, "google.com"은 유효한 웹 주소다.

### 2. DNS 조회

- 브라우저는 "google.com"의 IP 주소를 찾기 위해 DNS(Domain Name System) 조회를 수행한다. 이 과정에서 브라우저 캐시, OS 캐시, 라우터 캐시, ISP의 DNS 서버 등 여러 캐시 레벨을 확인하여 가능한 빠르게 주소를 찾으려고 한다.
- 만약 캐시된 정보가 없다면, DNS 서버는 도메인 이름을 IP 주소로 변환하는 요청을 처리한다.

### 3. TCP 연결

- IP 주소가 확인되면, 브라우저는 해당 서버와의 TCP(Transmission Control Protocol) 연결을 시작한다. HTTPS를 사용하는 경우, 이 단계에는 SSL/TLS 핸드셰이크도 포함되어 암호화된 연결을 구성한다.

### 4. HTTP 요청

- TCP 연결이 성공적으로 수립되면, 브라우저는 HTTP 프로토콜을 사용하여 웹 서버에 웹 페이지 데이터를 요청한다.

### 5. 서버 응답

- 서버는 브라우저의 요청을 받고 처리한 후, HTML 문서를 비롯한 필요한 데이터를 HTTP 응답으로 브라우저에 전송한다.

### 6. 페이지 렌더링

- 브라우저는 받은 HTML 문서를 파싱하고, DOM(Document Object Model) 트리를 구축한다.
- CSS 파일은 CSSOM(CSS Object Model) 트리를 구축하는 데 사용된다.
- 자바스크립트 파일은 파싱되어 실행된다. 자바스크립트는 DOM을 조작할 수 있으며, 페이지의 동적 변경을 가능하게 한다.
- DOM과 CSSOM은 결합하여 렌더 트리를 형성하고, 이를 바탕으로 페이지가 화면에 그려진다.

### 7. 추가 자원 로딩

- HTML 문서 내에서 참조된 이미지, 스타일시트, 스크립트 등의 외부 자원들이 로딩된다. 이 자원들은 페이지의 완성도를 높이기 위해 필요하다.

### 8. 사용자와의 상호작용

- 페이지가 완전히 로드되고 난 후, 사용자는 페이지와 상호작용할 수 있다. 이 과정에서 발생하는 이벤트들은 자바스크립트를 통해 처리될 수 있다.

이 과정은 대략적인 순서를 설명한 것으로, 실제로는 더 많은 최적화와 복잡한 단계가 포함된다. 예를 들어, 최신 웹 브라우저는 가능한 한 빨리 콘텐츠를 사용자에게 보여주기 위해 리소스를 병렬로 다운로드하고, 스크립트 실행을 최적화하는 등 다양한 기술을 사용한다.
