### SSR의 장점

**1. 초기 페이지 로드 성능**

![CSR](/static/images/csr.webp)

CSR (no pre-rendering in plain React app)

  - 서버가 빈 HTML 페이지를 클라이언트에 보내고, 브라우저가 HTML코드를 다운로드하여 화면을 렌더링한다.

  - 서비스에서 필요한 데이터를 클라이언트에서 추가로 요청하여 재구성해야 하기 때문에 전제적인 페이지 완료 시점은 SSR보다 느려진다.

  - 네트워크 상황이 좋지 않다면 CSR을 이용할 경우 사용자는 컨텐츠를 보기 전에 상당 시간 로딩 화면을 봐야 할 수도 있다.

![SSR](/static/images/ssr.webp)

SSR (pre-rendering)

  - 반면 SSR은 서버가 요청된 페이지를 처리하여 바로 렌더링할 수 있는 HTML 페이지를 브라우저에 보낸다. 즉, 서버에서 데이터를 요청하여 사용자에게 보여줄 HTML 페이지를 구성한다. 

  - 따라서, 사용자에게 보여지는 콘텐츠가 완료되는 시점이 빨라진다.

**2. 검색 엔진 최적화 (SEO)**

     검색 엔진이 완전히 렌더링된 페이지 콘텐츠를 크롤링하고 인덱싱할 수 있게 한다. 검색 엔진 크롤러는 종종 JavaScript 기반 콘텐츠에 어려움을 겪기 때문에, 페이지의 검색 가능성과 검색 순위를 향상시킬 수 있다.
