"Cascading"은 CSS(Cascading Style Sheets)에서 사용되는 핵심 개념 중 하나다. 이 용어는 스타일 규칙이 어떻게 여러 출처에서 하나의 요소에 적용되며, 이러한 규칙들이 어떻게 서로 상호 작용하는지를 설명한다. "Cascading"이란 용어 자체는 여러 스타일 규칙이 겹쳐질 때, 어떤 규칙이 최종적으로 적용될지 결정하는 과정을 의미한다.

CSS에서는 여러 스타일 규칙이 동일한 요소에 적용될 수 있으며, 이때 몇 가지 기준에 따라 어떤 규칙이 우선 적용될지 결정한다. 이러한 기준은 다음과 같다:

1. **중요도(Importance):** `!important`가 적용된 스타일 규칙이 가장 높은 우선순위를 가진다.
2. **명시도(Specificity):** 요소를 더 구체적으로 지정하는 규칙이 우선 적용된다. 예를 들어, ID 선택자는 클래스 선택자보다, 클래스 선택자는 태그 선택자보다 더 구체적으로 간주된다.
3. **소스 순서(Source Order):** CSS 코드 내에서 뒤에 위치한 규칙이 앞에 위치한 규칙보다 우선 적용된다. 명시도가 동일한 경우에 한한다.

이러한 규칙들은 서로 다른 스타일 시트에서 오는 규칙, 같은 스타일 시트 내에서의 규칙, 브라우저의 기본 스타일, 사용자 스타일 시트 등 여러 출처의 스타일 규칙들이 어떻게 결합되어 최종적으로 요소의 스타일을 결정하는지를 관리한다.

Cascading의 이러한 원리는 개발자가 복잡한 웹 페이지와 애플리케이션에서 일관된 스타일링을 보다 효율적으로 관리할 수 있게 해준다.
